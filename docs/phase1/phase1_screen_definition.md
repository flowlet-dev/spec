# Phase 1 MVP 画面一覧・画面責務定義

## 目的

本ドキュメントは、FLOWLET Phase 1 MVP における  
**画面構成と各画面の責務（何をする画面か）** を定義する。

- 技術スタック非依存
- Web / モバイルどちらにも転用可能
- API設計の土台となることを目的とする

---

## 設計方針

- 画面数は最小限
- CRUDベースではなく **業務操作ベース**
- 「1画面 = 1ユースケース責務」を原則とする

---

## 画面一覧

| 画面ID | 画面名 | 主な責務 |
|------|------|------|
| S-01 | ダッシュボード | 今月の収支・口座残高の俯瞰 |
| S-02 | 取引一覧 | 取引履歴の確認 |
| S-03 | 取引登録 | 収入・支出・振替の登録 |
| S-04 | 口座一覧 | 口座残高の確認 |
| S-05 | 口座詳細 | 口座内訳（目的含む）の確認 |
| S-06 | 目的管理 | 目的の作成・編集 |

---

## 画面責務定義

### S-01：ダッシュボード

#### 責務
- 今月の収支を即座に把握できる
- 現在の全口座残高を一覧で確認できる

#### 表示内容
- 今月の収入合計
- 今月の支出合計
- 今月の収支差額
- 口座ごとの現在残高

#### 備考
- 登録・編集操作は行わない
- MVPではグラフ表示は必須ではない

---

### S-02：取引一覧

#### 責務
- 登録済みの取引を時系列で確認する

#### 表示内容
- 取引日
- 取引種別（収入 / 支出 / 振替）
- メモ
- 金額
- from → to（表示用に整形）
- 収支計算対象フラグ

#### 操作
- 月切替
- 取引詳細の参照（MVPでは編集・削除は必須ではない）

---

### S-03：取引登録

#### 責務
- 収入・支出・振替を登録する

#### 入力項目
- 取引種別
- メモ
- 金額
- from（支出・振替時）
- to（収入・振替時）
- （任意）目的
- 取引日
- 収支計算対象フラグ

#### 業務ルール
- 種別により入力項目が動的に変化
- 目的は目的別管理有効口座のみ選択可能

---

### S-04：口座一覧

#### 責務
- すべての口座を一覧で確認する

#### 表示内容
- 口座名
- 現在残高
- 目的別管理有無

#### 操作
- 口座詳細画面への遷移

---

### S-05：口座詳細

#### 責務
- 特定口座の内訳を確認する

#### 表示内容
- 口座名
- 現在残高
- （目的別管理有効時）
  - 目的ごとの残高一覧

#### 備考
- MVPでは口座編集機能は必須ではない

---

### S-06：目的管理

#### 責務
- 目的（Purpose）を管理する

#### 操作
- 目的の作成
- 目的名の編集
- 初期割当金額の設定

#### 制約
- 目的は目的別管理有効口座にのみ作成可能
- 目的削除は MVP では必須ではない

---

## 画面遷移概要（簡易）

```
ダッシュボード
├─ 取引一覧
│    └─ 取引登録
├─ 口座一覧
│    └─ 口座詳細
│         └─ 目的管理
```

---

## Phase 1 でやらないこと

- 画面カスタマイズ
- 高度な検索・フィルタ
- グラフ・分析機能
- 認証・ユーザー管理
